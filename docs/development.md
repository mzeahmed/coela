# Development

This document describes the local workflow for contributing to Coela itself. It covers Go tooling only — the requirements for running a project *generated by* Coela (Docker, Composer, mkcert) are listed in the [README](../README.md).

A `Makefile` at the repo root wraps every command below (and a few release-only ones — see [Release Process](release.md)). Run `make help` to list them all; each section here names its shortcut.

## Requirements

- Go, matching the version declared in `go.mod`

## Format

```bash
go fmt ./...
```

Shortcut: `make fmt`

Formats every Go source file according to the standard Go style. Run this before committing.

## Static Analysis

```bash
go vet ./...
```

Shortcut: `make vet`

Checks the code for constructs the compiler does not flag, such as suspicious format strings or unreachable code.

## Tests

```bash
go test ./...
```

Shortcut: `make test`

Runs the test suite. Coela does not have automated tests yet; running this command still confirms every package compiles.

`make check` runs `fmt`, `vet`, and `test` together — this is what [Release Process](release.md) runs before tagging.

## Dependency Cleanup

```bash
go mod tidy
```

Shortcut: `make tidy`

Adds missing dependencies and removes unused ones from `go.mod` and `go.sum`. Run this after adding or removing an import.

`make update` runs `go get -u ./...` followed by `go mod tidy`, to upgrade every dependency to its latest version.

## Build

```bash
go build -o bin/coela
```

Shortcut: `make build`

Compiles a standalone binary at `bin/coela`.

## Run Without Building

```bash
go run . new
```

Shortcut: `make run`

Compiles and runs the CLI in a single step. Useful while iterating on a command.

Generated projects land under `tmp/` (gitignored) instead of the repo root, so iterating on `coela new` doesn't scatter projects across the coela source tree. This only applies to source builds: a released binary always generates in the current working directory (see `cmd.devOutputDir` and its `-X` override in `.goreleaser.yaml`).

`make help-cli` runs `go run . --help` to display Coela's own help text without scaffolding anything.

## Local Install

```bash
go install
```

Shortcut: `make install` (builds via `make build`, then installs `bin/coela` into `/usr/local/bin` with `sudo`, rather than Go's own `$GOBIN`)

Builds the binary and installs it into your `$GOBIN` (or `$HOME/go/bin` by default), making `coela` available on your `PATH`.

## Utilities

- `make clean` — removes `bin/` and `dist/`
- `make doctor` — prints the versions of the local toolchain (Go, Docker, Docker Compose, git, GoReleaser), useful to confirm everything required is installed
- `make version` — runs the installed `coela --version`
