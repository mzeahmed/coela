# Project Structure

This document describes the structure of a project generated by `coela new`, using a project named `project` as an example.

```
project/
├── app/
├── certs/
├── docker/
│   ├── nginx/
│   └── php/
├── docs/
├── traefik/
├── docker-compose.yml
├── Makefile
├── README.md
├── .env
└── .gitignore
```

## Folders

| Path | Role | Generated |
|---|---|---|
| `app/` | The framework itself, installed by Composer (e.g. Symfony's `symfony/skeleton`). | Always, by the stack's `Install` step. |
| `docker/nginx/`, `docker/php/` | Nginx and PHP-FPM configuration used by `docker-compose.yml`. | Always, from the stack's templates. |
| `traefik/` | Traefik's static and dynamic configuration (`traefik.yml`, `dynamic.yml`). | Always rendered, but only meaningful if Traefik is enabled — see note below. |
| `certs/` | The local TLS certificate and key generated by mkcert. | Only if the wizard's Traefik question was answered yes. |
| `docs/` | Space reserved for the generated project's own documentation. | Not generated by Coela — this is a convention for you to fill in, not a scaffolded folder. |

## Files

| Path | Role |
|---|---|
| `docker-compose.yml` | Defines every service (PHP, Nginx, the selected database, and optionally Traefik, Mailpit, Redis). |
| `Makefile` | Shortcuts for common operations (`make up`, `make down`, `make bash`, database migrations, etc.). |
| `README.md` | Project-specific instructions generated from the stack's `README.md.tmpl`. |
| `.env` | Database credentials (`DB_DATABASE`, `DB_USER`, `DB_PASSWORD`, ...) referenced by `docker-compose.yml` via `${...}` substitution. Gitignored. |
| `.gitignore` | Ignores `app/vendor`, `app/var`, `.env`, generated certificates, and IDE folders. |

## A Note on What Is Always Generated

`internal/scaffold.Generate` renders every `.tmpl` file it finds in a stack's template directory — it has no mechanism to skip a whole file based on a wizard answer. This means files like `traefik/traefik.yml` and `traefik/dynamic.yml` are always written, even if you answered "No" to the Traefik question; in that case `docker-compose.yml` simply has no `traefik` service to use them. See [templates.md](templates.md) for how conditional *content* within a single file works.
